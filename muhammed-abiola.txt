## Questions

###    1. What is Semantic HTML? 

Semantic HTML is a way of writing HTML that establishes the importance of the meaning of content, as opposed to just its appearance, providing greater accessibility
to various web elements.

###    6. What do we put in the head of our HTML document?

###    7. What is an id? 

An id is a html attribute used within an element to point to a unique ID in css or javascript.

###    8. What elements can I add an id to?

###    10. What is a class? 

A class is a html attribute used within an element to point to a class selector in css, or javascript to manipulate an element or group of elements with the class
attribute attached to them.

###    16. What is the difference between a section and a div

Semantics. A section behaves like a div, except for the fact that it is semantically meaningful to the reader. In fact, there is no behaviour the section performs that
is computationally different, but it may be targetted by search engines specifically as a section, as opposed to a div which has broad use that may not be to section off
some text.

###    34. Why should I use version control? 

version control is useful when tracking each change a programmer on a team makes. when dealing with complicated code maintained by multiple programmers, it's easy
to encounter problems with complexity. in using version control such as git, it makes the discovery and maintenance of code releases much more seamless and integrated.
this is less important when working alone, but still useful when tracking changes over time, and adding new members to a team.
